Learnings: Tool descriptions are crucial. LLMs love structured markdown. APIs aren't always meant to be wrapped directly.

github.com/NimbleMarkets/dbn-go/cmd/dbn-go-mcp github.com/AgentDank/dank-mcp github.com/ConAcademy/buttplug-mcp
MCP - Model Context Protocol - lets you expose data and tools to LLMs in a standardized way. dank-mcp was the fun one: structured registry data loaded into DuckDB, exposed through MCP. You could ask Claude questions about the data and it would write SQL queries. The big lesson: how you describe your tools matters more than how powerful they are. LLMs work best with well-structured interfaces.

Multimodal Debugging

Full debugging cycle from a screenshot

Simple input, harnessing:

  • Multi-modal capability
  • World understanding
  • Tool calling
  • Agentic looping

"A friend discovered a bug in this. Look at the attached picture, figure out what the bug is and correct it."

See image ‚Üí Identify bug ‚Üí Create fix ‚Üí Write test ‚Üí Run in Chrome ‚Üí Screenshot ‚Üí Report

github.com/ConAcademy/proving_parallelograms
A friend saw my parallelogram app and sent a screenshot of a bug - it was drawing a bowtie instead of a proper parallelogram. What blew my mind: Claude could see the image, understand the geometric error, generate a fix, create a test case, manipulate the page in Chrome to verify it, screenshot the result, and give me a report. End-to-end multimodal debugging.

Image ‚Üí Swift Code ‚Üí 3D Model ‚Üí Image...

  • Discovered Cadova, a Swift library for parametric, 3D CSG modeling
  • Realization: Swift as ground truth for 3D models
  • Like how Claude generates DOCX or SVG... this is Swift code ‚Üí 3MF model
  • Documented in WeaselToonCadova repo

Started with Claude Desktop, then evolved to human-less agentic looping with Claude Code

github.com/tomasf/Cadova github.com/ConAcademy/WeaselToonCadova
Discovered Cadova on HackerNews - a Swift library for parametric 3D CSG modeling that outputs 3MF files. The realization: Swift code can be the ground truth for 3D models, just like Claude generates SVG or DOCX. Started experimenting in Claude Desktop, then evolved to fully autonomous agentic loops with Claude Code - generate Swift, compile, render, screenshot, analyze, iterate. The WeaselToonCadova repo documents this workflow.
