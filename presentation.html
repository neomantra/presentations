April 2025

MCP Exploration ü§ñ

  • Built MCP servers: DataBento, buttplug-mcp
  • Culminated in dank-mcp
  • Cannabis dataset queries via DuckDB + Claude

Learnings: Tool descriptions are crucial. LLMs love structured markdown. APIs aren't always meant to be wrapped directly.

github.com/AgentDank/dank-mcp modelcontextprotocol.io
MCP - Model Context Protocol - lets you expose data and tools to LLMs in a standardized way. dank-mcp was the fun one: Connecticut cannabis registry data loaded into DuckDB, exposed through MCP. You could ask Claude questions about products and it would write SQL queries. The big lesson: how you describe your tools matters more than how powerful they are. LLMs work best with well-structured interfaces.

  • Developed screentime-mcp
  • Agentic loops: generate ‚Üí test ‚Üí improve
  • Used Charmbracelet Crush to develop MCPs...
  • ...while improving Crush's MCP support

World knowledge + SQL = deterministic answers to iterate on

github.com/AgentDank/screentime-mcp
This is where things got meta. I was building MCP servers while using an MCP-enabled tool (Crush) that I was also contributing to. The key insight: LLMs have world knowledge but sometimes you need deterministic answers. SQL gives you that ground truth, then you can iterate with the LLM to refine queries and visualizations.

Multimodal Debugging

  • Friend pasted error: bowtie shape instead of parallelogram
  • Fully automated pipeline:

See image ‚Üí Identify bug ‚Üí Create fix ‚Üí Write test ‚Üí Run in Chrome ‚Üí Screenshot ‚Üí Report

github.com/ConAcademy/proving_parallelograms
A friend saw my parallelogram app and sent a screenshot of a bug - it was drawing a bowtie instead of a proper parallelogram. What blew my mind: Claude could see the image, understand the geometric error, generate a fix, create a test case, manipulate the page in Chrome to verify it, screenshot the result, and give me a report. End-to-end multimodal debugging.

Swift ‚Üí 3D Model

  • Discovered Cadova on HackerNews
  • Swift library for parametric 3D modeling ‚Üí 3MF output
  • Realization: Swift as ground truth for 3D models
  • Like how Claude generates DOCX or SVG...
  • Documented in WeaselToonCadova repo

Started with Claude Desktop, got serious with Claude Code

github.com/tomasf/Cadova github.com/ConAcademy/WeaselToonCadova
I was mid-comment on a HackerNews post about Cadova when it clicked. Cadova lets you write Swift code that outputs 3MF files - 3D printable models. Just like Claude can generate SVG for images or DOCX for documents, it could generate Swift code as a deterministic representation of 3D models. The WeaselToonCadova repo documents the whole agentic loop: generate code, render, analyze, iterate.
